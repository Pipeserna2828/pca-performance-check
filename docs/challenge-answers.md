# Respuestas a las preguntas del reto

## 1. ¿Qué problema organizacional resuelve tu herramienta?

Resuelve el retrabajo que se genera cuando las solicitudes de performance llegan incompletas, mal priorizadas o sin condiciones mínimas para ejecutarse. PCA Performance Check estandariza el intake, mejora la priorización y ayuda a decidir si vale la pena entrar al flujo de performance.

## 2. ¿Cómo evitarías que se convierta en una herramienta aislada?

Separando un núcleo común reusable, contratos estándar de entrada y salida, una API estable y reglas desacopladas del frontend. Eso permite que otros módulos de calidad reutilicen el mismo flujo de registro, ejecución y consulta de resultados.

## 3. ¿Qué mejoras agregarías si tuvieras más tiempo?

Agregaría autenticación corporativa, persistencia administrada, dashboard histórico, reglas configurables desde base de datos, integración con Azure DevOps, observabilidad centralizada y medición de calidad del uso de IA.

## 4. ¿Cómo escalaría esta solución a toda la empresa?

La escalaría como un capability interno de Quality Engineering con un núcleo común y módulos por especialidad. El mismo patrón puede soportar seguridad, calidad funcional y analítica, siempre con contratos estándar y gobierno técnico central.

## 5. ¿Dónde usarías IA y dónde no?

Usaría IA para explicar resultados, redactar hallazgos y proponer siguientes pasos. No usaría IA para calcular scores, validar reglas críticas ni tomar la decisión base del análisis, porque esas decisiones deben ser trazables, consistentes y auditables.
